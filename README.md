How do I interpret user needs and implement them into a program? How does creating user stories help with this?

One thing I learned in this course is that understanding user needs starts with listening to what the user wants the program to do. User stories helped me break larger requirements into smaller pieces that were easier to understand and work on. They kept the focus on the user instead of just the technical side of development.

How do I approach developing programs? What Agile processes do I hope to incorporate into my future development work?

I usually approach development by breaking larger projects into smaller tasks so they feel more manageable. This course showed me how useful Agile practices can be for staying organized and adapting to changes. In future projects, I would like to continue using sprint planning, regular feedback, and retrospectives because they help teams stay on track and improve throughout development.

What does it mean to be a good team member in software development?

To me, being a good team member means communicating, being dependable, and being willing to help others when needed. It also means listening to different ideas and being open to feedback. Throughout this course, I learned that teamwork is just as important as technical skills when working on a software project.
